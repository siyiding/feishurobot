# 整车测试助手飞书机器人

飞书机器人，用于整车测试场景，支持缺陷查询、报告生成等能力。

## 功能模块

| 模块 | 描述 | 状态 |
|------|------|------|
| 模块1 | 飞书机器人基础（消息接收/意图识别/路由分发） | ✅ M1 Week1-2 |
| 模块2 | 飞书项目数据对接（缺陷/任务） | ✅ M1 Week2 |
| 模块3 | 飞书电子表格用例库 | 🚧 M2 |
| 模块4 | 通知推送系统 | 🚧 M1 Week4 |
| 模块5 | 自然语言问答+报告生成 | 🚧 M3 |
| 模块6 | DR平台对接 | 🚧 M3 |

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 配置环境变量

创建 `.env` 文件：

```bash
FEISHU_APP_ID=cli_xxxxxxxxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxxxxxxxxxxx
FEISHU_BOT_NAME=整车测试助手
DEBUG=true
LOG_LEVEL=INFO
```

### 3. 运行服务

```bash
# 开发模式
python main.py --reload

# 生产模式
python main.py
```

服务启动后访问：
- API文档：http://localhost:8000/docs
- 健康检查：http://localhost:8000/health

### 4. 飞书应用配置

#### 4.1 创建飞书应用

1. 打开 [飞书开放平台](https://open.feishu.cn/app)
2. 点击「创建企业自建应用」
3. 填写应用名称（如：整车测试助手）和描述
4. 获取 `App ID` 和 `App Secret`，填入 `.env`

#### 4.2 配置机器人能力

1. 在应用详情页，点击「添加应用能力」
2. 搜索「机器人」，点击启用
3. 配置机器人信息：
   - **机器人名称**：`整车测试助手`
   - **机器人描述**：整车测试助手，帮你查缺陷、生成报告
   - **应用图标**：可上传图标（建议尺寸 256x256）

#### 4.3 配置权限

在「权限管理」中添加以下权限：

| 权限名称 | 权限标识 | 用途 |
|---------|---------|------|
| 获取应用授权 | `bot` | 机器人基础能力 |
| 查看、评论和编辑文档 | `docx:document:rw` | 报告生成 |
| 查看电子表格 | `sheets:spreadsheet:readonly` | 用例库读取 |
| 编辑电子表格 | `sheets:spreadsheet:write` | 用例库写入 |
| 查看项目列表 | `project:project:readonly` | 项目列表 |
| 查看任务 | `project:task:readonly` | 任务查询 |
| 创建/编辑任务 | `project:task:write` | 缺陷管理 |

#### 4.4 配置事件订阅

1. 在「事件订阅」中，添加以下事件：
   - `im.message.receive_v1` - 接收消息

2. 配置请求地址：
   ```
   https://your-domain.com/webhook/feishu
   ```

#### 4.5 配置可用范围

- 测试阶段：设为特定测试人员
- 正式上线后：可扩大至全部员工

#### 4.6 发布应用

1. 在「版本管理与发布」中创建版本
2. 填写版本说明
3. 提交审核（如有）
4. 发布上线

### 5. 机器人配置说明

#### 5.1 修改机器人名称

在飞书开放平台的应用详情页「机器人」配置中，可以直接修改机器人名称。

#### 5.2 修改机器人头像

1. 在「应用信息」中，点击应用图标
2. 上传新图标（建议 PNG 格式，256x256 像素）
3. 保存后生效

#### 5.3 配置机器人介绍

在飞书开放平台可以设置机器人的欢迎语和快捷操作：
- 欢迎语示例：「您好！我是整车测试助手，可以帮您查询缺陷、生成报告等。」
- 快捷回复：可配置常见问题答案

## 意图类型

| 意图类型 | 触发关键词 | 示例 |
|---------|-----------|------|
| QUERY（查询） | 查、查询、有没有、多少、列表 | 查ICC项目缺陷 |
| ACTION（操作） | 创建、新建、更新、修改、关闭 | 创建缺陷 |
| REPORT（报告） | 报告、周报、月报、统计、汇总 | 生成本周周报 |

## 架构决策

详见 [docs/p0-decisions.md](docs/p0-decisions.md)：

- **D1**: Bitable并发写入策略 - 机器人只写白名单字段
- **D2**: 对话上下文持久化边界 - 30分钟快照+三重保护
- **D3**: 飞书机器人鉴权 - M1-M2白名单模式
- **D4**: M2验收标准 - 字段完整性+5项抽查机制
- **D5**: DR平台API策略 - 定时+实时混合+10分钟Buffer
